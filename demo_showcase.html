📐

STEP 拓扑解析 核心

基于 OpenCASCADE 解析 STEP 文件拓扑结构，提取 PLANE/CYLINDER/CONE/SPHERE/TORUS 面类型，生成像素级标签。拓扑类型提取准确，但孔/螺栓等语义分类依赖启发式规则，对复杂零件可能需调参

🎯

6DoF Ground Truth

每帧输出精确的相机位姿（旋转矩阵+平移向量+四元数），BOP 格式标准输出，直接用于 6DoF 位姿估计训练

🎨

Instance Segmentation 新增

RGB Mask 编码 (InstanceID, FeatureTypeID, FeatureIndex)，支持同类多实例区分，如多个螺栓/孔分别标注

🌊

Sim-to-Real 增强

高斯噪声 / 运动模糊 / 随机遮挡 / 色彩抖动 / 深度噪声，弥合仿真-现实差距，提升模型泛化能力

📏

线性深度图

OpenGL NDC → 线性深度转换，uint16 编码，支持 mm/m/cm/inch 单位，直接用于机器人抓取规划

🎲

Domain Randomization

随机光照 / 相机抖动 / 背景替换，大幅提升训练数据多样性，增强模型鲁棒性

⚡

C++ 高性能渲染

OpenGL 3.3+ + BVH 加速 + 零拷贝 STL 解析，约 0.5s/帧 (800x600)，批量生成效率高

📊

📊 数据质量报告

自动生成数据集质量报告（JSON+HTML），含训练就绪度评分、BOP合规检查、标注质量分析，可交付客户

🔄

批量 Pipeline Professional+

一键处理整个 STEP 文件目录，自动完成 拓扑→渲染→增强→标注→报告 全流程，支持大规模数据生成

🚀 快速启动
⚠️ 重要：完整数据生成需要编译 C++ 渲染引擎，未编译时仅可查看预生成的 Demo 数据

📋 系统要求

必须：Python 3.8+（推荐 Miniconda）

完整功能：Visual Studio 2019+（含 C++ 桌面开发工具）、CMake 3.16+

STEP 拓扑：cadquery（~500MB，含 OpenCASCADE）

⚠️ 没有 C++ 引擎 = 只能查看 Demo 数据，无法生成新数据

方式一：一键启动（查看 Demo）

Windows 用户双击 start_demo.bat，自动安装依赖并打开浏览器：

REM 自动检测 Python + 安装依赖 + 启动 Web UI start_demo.bat

💡 未编译 C++ 引擎时，进入 Demo 模式查看预生成样例数据

方式二：编译 C++ 引擎（解锁完整功能）

编译后可上传自己的 CAD 模型生成数据：

REM 自动检测 Visual Studio 并编译 build_and_compile.bat

编译成功后重新运行 start_demo.bat 即可

方式三：Docker 部署（推荐服务器/云端）

docker build -t huhb3d-synthetic . docker run -p 7860:7860 huhb3d-synthetic # 访问 http://localhost:7860
